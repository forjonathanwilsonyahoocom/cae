Pipeline / Linear: Agents operate strictly sequentially; maximizes execution speed and predictability but completely destroys system adaptability if upstream data drifts.

Anti-Pattern: Brittle Failure Cascades (The Degraded Form of Linear Topology)
    • Architectural Definition: Occurs when a pipeline of sequential agents lacks dynamic error-absorption barriers or upstream verification hooks. Data flows strictly in one direction, causing any upstream semantic drift or tool failure to pass down the chain unchecked.
    • Underlying Forces:
        1. Error Amplification: In a sequential system without isolation, a minor logical divergence at Turn 1 grows into a catastrophic hallucination or broken tool payload by Turn 5.
        2. Organizational Memory Loss: Downstream agents possess zero knowledge of the original execution context, blinding them to why an upstream payload looks mutated.
    • The "No-Planning" Symptom: Developers link agents sequentially (Agent A \(\rightarrow \) Agent B \(\rightarrow \) Agent C) assuming perfect, deterministic data schemas from highly variable model outputs.
    • System Breakdown State: Silent data corruption. The final agent confidently executes a completely broken, nonsensical payload generated by upstream drift, without ever triggering a system exception or a retry loop.
